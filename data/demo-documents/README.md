# Demo Documents

Sample documents bundled with the repo for the on-stage demos. Use these
after dropping them into the Observatory or via the demo runner.

## What's here

| File | Type | Purpose | Source |
| ---- | ---- | ------- | ------ |
| `nextera_quarterly_report.pdf` | Synthetic | OCR demo + cross-validation against the SQL `sales` table | Generated by `scripts/generate_ocr_demo_doc.py` from synthetic Nextera demo data |
| `snowflake-fy2025-first50.pdf` | Public (SEC) | OCR demo + structured extraction into the `competitors` table | First 50 pages of Snowflake's FY2025 Annual Report (Form 10-K), publicly filed with the SEC |
| `snowflake-fy2025-annual-report.pdf` | Public (SEC) | Full annual report for deeper OCR / search demos | Snowflake FY2025 Annual Report (Form 10-K), publicly filed with the SEC |
| `DEMO_QUERIES.md` | Doc | Curated query set per demo (Meridian Health ADR, OCR, structured extraction) | This repo |

## What's intentionally **not** here

- **No real patient data**, no real customer records, no real PII.
- The Nextera quarterly report is **synthetic** — `Nextera` is a fictional
  company invented for the talk. Regenerate via
  `python scripts/generate_ocr_demo_doc.py`.
- Third-party analyst reports (e.g. industry benchmark PDFs) are not
  redistributed here — only SEC filings, which are public record.

## Healthcare ADR demo

The "Meridian Health ADR" demo (see `DEMO_QUERIES.md`) refers to a
fictional EU healthcare ADR document. The actual PDF lives outside this
repo (it's a demo prop, not a redistributable asset). Drop your own
healthcare ADR PDF into the Observatory to follow along.
